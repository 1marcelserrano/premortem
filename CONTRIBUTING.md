# Contribuir

Obrigado pelo interesse. Esta skill é open-source sob licença MIT.

## Como propor mudanças

1. Faça um fork e crie um branch a partir de `main`.
2. A fonte de verdade do comportamento é `skills/premortem/SKILL.md`. Comece por lá.
3. Mantenha o frontmatter (`name`, `description`) intacto — ele é parseado pelos loaders.
4. Use Conventional Commits (`feat:`, `fix:`, `docs:`…).
5. Abra um PR descrevendo o que muda e por quê.

## Princípios

- **Voz direta.** Frases curtas, voz ativa, zero floreio. Sem léxico inflado, sem paralelismo negativo.
- **Não fabrique números.** Toda afirmação de eficácia cita uma fonte. A prova vem dos papers, não de marketing.
- **O mecanismo é o enquadramento.** "Isso já falhou, explique como" é o que faz a skill funcionar. Mudanças que diluem isso não entram.

## O que evitar

- Editar a página (`page/`) sem refletir a mudança na fonte da página.
- Commitar assets binários grandes sem necessidade.
- Tokens, `.env` ou credenciais — nunca.
